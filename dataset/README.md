## English Documentation
# DAQUAR Dataset Viewer Documentation

## 1. Overview

This static webpage is designed for browsing the **DAQUAR Dataset (Processed) for Visual Question Answering (VQA)**.

It displays images, questions, answers, and the answer space in one interface, making it easier to inspect the dataset structure, question patterns, answer distribution, and multi-answer examples.

The viewer is only a data inspection tool. It does not modify the original dataset and is not used during model training.

## 2. Dataset Source

This project uses the processed DAQUAR dataset from Kaggle:

https://www.kaggle.com/datasets/tezansahu/processed-daquar-dataset

DAQUAR is a classic indoor-scene visual question answering dataset. Its images are mainly based on NYU-Depth V2 indoor scenes. The task is to answer a natural language question based on the content of a given image.

## 3. Dataset Contents

The viewer uses the following files:

- `data_train.csv`: training question-answer pairs
- `data_eval.csv`: evaluation question-answer pairs
- `answer_space.txt`: 582 candidate answer classes
- `images/`: dataset images
- `daquar_viewer_data.js`: static data bundle generated from the CSV files and answer space
- `index.html`: main viewer page

Dataset integrity check:

- Training set: 6795 QA pairs, 794 images
- Evaluation set: 5673 QA pairs, 653 images
- Total: 12468 QA pairs, 1447 images used by QA pairs
- Answer space: 582 classes
- All images referenced by the CSV files exist
- All primary answers can be mapped to `answer_space.txt`

## 4. How to Open the Viewer

Locally, open:

```text
index.html
```

If deployed with GitHub Pages, open the generated GitHub Pages URL, for example:

```text
https://your-username.github.io/daquar-viewer/
```

The page does not require a backend server because the CSV data has been bundled into `daquar_viewer_data.js`.

## 5. Viewer Features

### Train / Eval / All Split

The top buttons switch between:

- `Train`: training split
- `Eval`: evaluation split
- `All`: training + evaluation

### Image-Based Browsing

The left panel groups QA pairs by image. After selecting an image, the center panel shows:

- the image
- all questions for that image
- the primary answer for each question
- the full CSV answer if the original sample contains multiple answers

### Search

The top search box supports searching by:

- image id
- question text
- answer text

### Filter by Answer

The `Filter QA by answer` box filters QA pairs by answer, for example:

```text
books
```

It matches both primary answers and any answer in the raw CSV answer list.

You can also click an answer label in the Answer Space panel to filter by that answer.

### Answer Space

The Answer Space panel shows the 582 candidate answer classes.

- Green labels: answers that appear as primary answers in the current filtered view
- Gray/white labels: answers that exist in the 582-class answer space but do not appear in the current view

The order is preserved from the original `answer_space.txt`. This order corresponds to the class label index used in model training.

### Automatic Question Type

The viewer automatically infers a rough question type from the question text:

- Counting
- Color
- List / multiple
- Location
- Spatial
- Attribute
- Object
- Person
- Other

These types are heuristic labels generated by the viewer. They are not official DAQUAR annotations and are not part of the original CSV files.

## 6. Potential Issues in the Dataset

### Multi-Answer Samples Are Often Converted to Single-Label Targets

Some raw answers contain multiple valid answers, for example:

```text
clock, telephone, fire_extinguisher
```

Many baseline scripts use:

```python
answer.replace(" ", "").split(",")[0]
```

This keeps only the first answer, `clock`, as the classification target. This makes the task compatible with single-label 582-class classification, but it loses information.

For later experiments, it is useful to report both:

- strict / primary accuracy: only the first answer is treated as correct
- any-answer accuracy: any answer in the raw answer list is accepted as correct

### Spelling and Grammar Errors

Some original questions contain spelling or grammar errors, such as:

- `waht`
- `how manu`
- `what sre`
- `telephon`

The viewer preserves the original text and does not edit the dataset.

### Similar Answer Classes

The 582-class answer space contains many semantically similar labels, for example:

- `clock` / `alarm_clock`
- `book` / `books` / `bookshelf`
- `bin` / `garbage_bin`
- `chair` / `baby_chair`
- `soap` / `bar_of_soap` / `bottle_of_soap`

This makes exact-match accuracy strict: a semantically similar but different class is still counted as incorrect.

### Fine-Grained Indoor Reasoning

DAQUAR questions often involve:

- small object recognition
- spatial relations such as left, right, front, behind, above, and between
- counting
- multi-object listing

These are more difficult than ordinary image classification.

## 6. 当前数据集可能存在的问题

### 多答案样本被单标签化

部分样本的原始答案包含多个答案，例如：

```text
clock, telephone, fire_extinguisher
```

但常见 baseline 代码会使用：

```python
answer.replace(" ", "").split(",")[0]
```

也就是只取第一个答案 `clock` 作为分类标签。这样可以把任务转成单标签 582 类分类，但会丢失信息。

因此后续模型评估时可以同时报告：

- strict / primary accuracy：只命中第一个答案算正确
- any-answer accuracy：命中原始答案列表中的任意一个都算正确

### 数据中存在拼写和语法错误

DAQUAR 中有一些原始问题存在拼写错误或语法不规范，例如：

- `waht`
- `how manu`
- `what sre`
- `telephon`

viewer 保留这些原始文本，不会自动修改数据。

### 答案空间中存在相似类别

582 个答案中有一些非常相近的类别，例如：

- `clock` / `alarm_clock`
- `book` / `books` / `bookshelf`
- `bin` / `garbage_bin`
- `chair` / `baby_chair`
- `soap` / `bar_of_soap` / `bottle_of_soap`

这会让 exact-match accuracy 比较严格：模型回答语义相近但类别不同，也会被判错。

### 室内图像细节复杂

DAQUAR 问题经常涉及：

- 小物体识别
- 左右、前后、上下、between 等空间关系
- 计数问题
- 多物体列表问题

这些问题比普通图像分类更难。
